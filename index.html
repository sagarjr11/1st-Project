02

Component Breakdown

🔀
Route 53
DNS / Routing
Health-check-based DNS failover. Routes traffic to healthy endpoints; supports latency-based, geolocation, and weighted routing policies.
DNS failoverhealth checksTTL control
⚡
CloudFront
CDN / Edge
Caches static assets at 400+ edge locations. Terminates SSL, absorbs DDoS via Shield Standard, and drastically reduces origin EC2 load.
SSL terminationedge cacheDDoS
⚖️
ALB
Application Load Balancer
Distributes traffic across EC2 instances in multiple AZs using round-robin or least-connections. Performs deep health checks and deregisters unhealthy targets automatically.
multi-AZpath routinghealth checks
📈
Auto Scaling Group
EC2 / Compute
Maintains desired EC2 count across AZs. Scales out on CPU > 60% or ALB request count. Replaces failed instances automatically. Supports mixed On-Demand + Spot.
target trackinginstance refreshspot mix
🗄️
RDS Multi-AZ
Managed Database
Synchronous replication to a standby in a second AZ. Automatic failover in under 60 seconds with no app-side changes needed. Not a read replica — zero data loss.
auto failoversync replica<60s RTO
⚡
ElastiCache Redis
In-Memory Cache
Stores session state and hot query results so any EC2 instance can serve any request. Prevents session loss during instance terminations from Auto Scaling events.
session storequery cachecluster mode
📊
CloudWatch
Observability
Collects EC2, ALB, and RDS metrics. Alarms trigger SNS notifications and ASG scaling actions. Logs Insights queries for debugging across all instances.
metricsalarmslog insights
🛡️
AWS WAF
Security
Enforces OWASP Top-10 managed rules, rate limiting per IP, and geo-blocking. Attached directly to the ALB — malicious traffic is blocked before hitting EC2.
OWASP rulesrate limitgeo-block
03

Failure Modes & Handling

Failure Scenario Detection Automated Response
EC2 instance crash ALB health check fails (2× threshold) ASG terminates + launches replacement; ALB deregisters target
Full AZ outage ALB detects all targets unhealthy in AZ ALB routes 100% to healthy AZ; ASG scales up instances there
Traffic spike (3×) CloudWatch: CPU > 60% for 2 min ASG scale-out policy adds instances (~3 min warm-up)
RDS primary failure RDS health check + CloudWatch Multi-AZ standby promoted; CNAME updated — <60s downtime
Bad deployment (5xx spike) CloudWatch ALB 5XX alarm fires SNS alert + trigger ASG instance refresh rollback
DDoS / bot flood WAF rate-limit rule triggers WAF blocks IPs at edge; Shield Standard absorbs volumetric
